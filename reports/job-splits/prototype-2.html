Job Split Defaults

When a job has multiple technicians assigned, how should revenue be distributed? This default applies to all completed jobs. Admins can override per job as needed.

Default split method
All to First Assigned
100% of job revenue is attributed to the first technician added to the job. Subsequent technicians receive 0%.
e.g., Marcus Webb: 100% / Jamie Torres: 0%
Equal Split
Revenue is divided evenly among all technicians assigned to the job at completion.
e.g., 2 techs: 50% / 50%  |  3 techs: 33% / 33% / 33%
Proportional to Time Logged
Revenue is split based on the number of hours each technician clocked on this job. Requires time tracking to be enabled.
e.g., Marcus 5.5h (69%) / Jamie 2.5h (31%) on an 8h job
Manual per Job
No default is applied. Admins must manually set the split on each job before it counts in reports.
Splits will show as "Pending" until set
⚠ Changing this default will only affect new jobs going forward. Existing completed jobs with splits already set won't be affected unless manually updated.
ⓘ Admins and owners can always override the split on any individual job. Field technicians can only see their attributed revenue in the leaderboard — not the split percentages.
